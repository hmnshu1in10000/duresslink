Info & Tutorial

What is SecureNote?

SecureNote (internal name: DuressLink) is a zero-knowledge, client-side-only dual-password link protection tool. It allows you to encrypt a sensitive target URL behind a real password, while simultaneously encrypting an innocent decoy URL behind a duress password — both packed into a single link.

Why Use It? (Plausible Deniability)

If you are forced or coerced by an adversary, inspector, or employer to unlock your shared link, you can safely hand over your duress password. This decrypts and redirects the visitor to an innocent decoy website (like a random Wikipedia page).

Because there is only one password field on the receiver page and the two encrypted payloads are shuffled randomly, it is mathematically impossible for an outside observer to prove that a second "real" password or target URL exists.

How It Works (The Cryptography)

SecureNote operates entirely in the browser using the Web Cryptography API. No plaintext URLs or passwords are ever sent to a server.

Key Derivation Passwords are hashed using PBKDF2 with SHA-256 and 310,000 iterations to derive high-strength 256-bit AES-GCM encryption keys.
Payload Encryption Both URLs are encrypted independently using AES-GCM (256-bit). Random 12-byte initialization vectors (IVs) are created for each.
Shuffled Packing The two resulting encrypted payloads (Blob A & B) are randomly shuffled so their order doesn't leak which is the real URL, and packed in the hash fragment.
Redirection & History The receiver page attempts parallel decryption. Once unlocked, the script purges the hash fragment from browser history using history.replaceState before redirecting.

Step-by-Step Tutorial

  • 1 Vault Setup (Optional)
    When you first load the generator, you can choose a 4-digit PIN to set up your Local Vault. This vault stores your settings, custom decoy URLs, and your default duress password locally in your browser, encrypted using PBKDF2 (200,000 iterations).
  • 2 Enter Target & Password
    Paste the sensitive link you want to protect in the "Your Link" field. Then, choose a strong password (minimum 8 characters) that the real recipient will use to open the note.
  • 3 Configure Decoy Settings
    Select what the decoy link will open if the duress password is entered. By default, it automatically fetches a random live Wikipedia article, but you can edit it manually or select from your custom saved decoys list.
  • 4 Configure Duress Password
    Set the password that triggers the decoy redirection. You can use your saved duress password (default is open) or type in a unique one-time duress password for a single link.
  • 5 Shorten & Generate
    Click **Generate Secure Link** to create the classic unshortened URL. Alternatively, toggle **Shorten link** to securely map the long URL to a short code using the is.gd API. The generated link keeps your custom domain name (e.g., /open/#xyz) and resolves the code client-side in the background.

Key Security Features

Plausible Deniability Dual payloads randomly ordered, giving an adversary no way to verify if a second password/URL exists.
CORS-Safe Decoys Auto-Wikipedia prefetching fetches clean, live random articles via origin-isolated API calls.
Permanent Shortening Shortened links utilize the is.gd API as a zero-auth, permanent lookup key-value store, retaining your custom domain name and performing resolution client-side.
History Leakage Defense Clears location hash from history state immediately before redirecting, leaving no browser traces.
Brute-Force Friction Failed password attempts trigger a uniform 800ms delay to thwart bot attacks.
Scrubbed Memory Passwords and cryptographic keys are zeroed out from variables immediately after use.
